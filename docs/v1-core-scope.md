# Core Module Scope

> Canonical customer-facing core scope for Bantu Niaga modules.
> Core features must be useful for solo entrepreneurs, micro SMEs, and growing SMEs without making the product feel like a heavy ERP.

---

## Packaging Rule

Core features are for the daily workflow:

- **Record** the business activity.
- **Organize** the information.
- **Share** the output with customers, staff, or suppliers.
- **Track** basic status and progress.

Add-on features are for premium growth:

- **Automate** repeat work.
- **Analyze** performance.
- **Integrate** with outside platforms.
- **Approve** sensitive actions.
- **Predict** what needs attention.
- **Scale** across more staff, stock, locations, or compliance needs.

This keeps the base product valuable without giving away every future upsell.

## Module 1 — Admin

Core features:

- Digital document storage.
- Business profile document folder.
- Receipt, contract, licence, and HR document folders.
- Secure file sharing link.
- Simple task board: To Do, Doing, Done.
- Task due dates and assignees.
- Basic notification feed.
- Compliance reminder calendar.
- Basic document templates.
- Internal notes.

Keep out of core:

- Digital signature.
- Advanced custom document builder.
- Auto document categorization.
- Approval workflow.
- Audit trail report.
- Smart compliance alerts.
- Extra storage packages.

## Build Implication

Build the simple loop first for each module. A module is ready for customers when an owner can complete the common daily action without training:

- Admin: store and share a document, then track a task.
- Finance: create an invoice, share it, and mark it paid.
- Operations: create a product/service, track an order, or add a booking.
- Sales: capture a lead or take a simple POS sale.
- HR: create an employee profile and record leave.
- Marketing: add customers, segment them simply, and prepare a campaign.

More advanced features should be placed in `docs/marketplace-addons.md` unless they are required to make the daily loop work.
